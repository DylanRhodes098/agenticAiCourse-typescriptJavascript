G - Goals / Instructions : What do you want it to do?
    - Persona : Character of llm (eg customer centric approach)
    - Rules : Things it can and can not do
    - Process : Mandatory steps that need to be followed (eg step 1 -> X, step 2 -> Y)

A - Actions : The llms ability to interact with the external enviornment to gather data and output an accurate reponse

I - Infomration : Tools that decide the next action to ouput the most accuarte response
    - Task : 
    - History : Accumilation of previous actions and llm outputs 
    - Inputs : User requests 
    - Environment state : 

L - Language : How do we want it to communicate with us?
